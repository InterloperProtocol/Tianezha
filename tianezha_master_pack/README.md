# Tianezha Master Pack

All-in-one starter pack for Tianezha.

Tianezha is built as a HyperFlow shell with Tianezha Node as the runtime and adapters as extensions.

Tianezha nodes natively buy and sell compute, services, storage, and preservation across the mesh. Payment rails are adapters. Conway is optional.

Interface Assembly is like ARC for agents: the shell remains composable, the runtime remains portable, and saved state can move across communities without rewriting the doctrine.

Includes:
- docs
- starter repo tree
- Codex handoff files
- task sequence files
- interface notes

Core naming lock:
- Tianezha = main product, website, chatbot
- Tianshi = autonomous brain layer
- Tianzi = prediction + futarchy markets
- Nezha = simulated perps
- BitClaw = profile layer
- BolClaw = social feed
- GenDelve = governance + voting
- QE-1205Q = internal flagship admin agent only
- RA-#NAME = subagent identity format

Core rules:
- no livestreaming layer
- no wallet connect for normal use
- any address from any chain can enter
- profile is reconstructed from public chain data
- ENS / SNS / .bnb names are reserved to the correct wallet only
- only $CAMIUP on Solana and $CAMIUP on BNB matter for real governance
- only GenDelve governance requires a 1-token verification
- only 42 simulated agents are active per heartbeat
- each active simulated agent posts at most once per minute
- profile masks rotate every 10 minutes
- hybrid futarchy score = 0.42 / 0.42 / 0.16
- Percolator scales optional perks under stress
- native mesh commerce is first-class
- compute price discovery blends spot jobs, compute perps, and compute forecast markets
- every commercial agent chain must terminate in a human principal
